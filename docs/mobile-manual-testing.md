# モバイル手動E2E確認手順

`apps/mobile`の各画面は`__DEV__`限定のモックログイン（ログイン画面下部の「[DEV] モックユーザーでタブ画面を確認」ボタン）でUIの見た目は確認できるが、モックセッションはSupabaseに実在しないため**実際のDB読み書きは検証できない**（RLSに弾かれる、または常に0件になる）。フォーム送信・イベント参加登録・チャット送受信・写真アップロードなど、実際にデータを書き込む処理は、本番Supabaseに対して実アカウントでの手動確認が必要。

## 準備

```bash
cd apps/mobile
npx expo start --ios   # シミュレータは起動後 `i` キー、実機はExpo GoアプリでQRコードを読み取る
```

## 確認したい流れごとの手順

### ① 初期登録フォーム〜承認
1. モバイルの「アカウントをお持ちでない方はこちら」から**テスト用メールアドレス**でサインアップ（普段使わないメール、または`+test1`のようなエイリアス）
2. 届いた確認メールのリンクをタップ→メール確認完了
3. モバイルでログイン→`InitialRegistrationScreen`が表示されるので実際に入力して送信
4. Web管理画面（`/admin-xxxx`）にPCブラウザでログインし、承認待ち一覧にテストユーザーが出ているか確認→承認
5. モバイルに戻りアプリを再起動→`fully_active`になりタブ画面に遷移するか確認

### ② Passport
- ①のユーザーで、まだイベント参加していない状態のPassport表示を確認（Journey Stampsが空状態になっているか）

### ③ Journey（イベント参加）
1. Web管理画面で軽いテスト用イベントを1件作成（本番イベントと紛れないよう「テスト」と分かるタイトルで）
2. モバイルのJourneyタブでそのイベントに実際に参加登録
3. Web管理画面のイベント参加者一覧に反映されているか確認
4. Passportタブに戻りJourney Stampsが1つ増えているか確認

### ④ Truss Embassy（チャット）
1. モバイルからテストメッセージを送信
2. Web管理画面のAdminChatで同じメッセージが届いているか確認（運営受信箱システムIDの実装も含めて一通り確認できる）
3. Web側から返信→モバイルにリアルタイムで届くか確認（アプリを開いたまま待つ）

### ⑤ Memories（写真アップロード）
1. モバイルから③のテストイベントに紐づけて写真を1枚アップロード
2. Web管理画面のギャラリー承認画面で承認
3. モバイルのMemoriesタブに表示されるか確認

## 後片付け

検証が終わったら、Web管理画面からテストユーザー・テストイベント・テスト写真を削除しておく（本番データに混ざったままにしない）。

## 別案: ステージング用Supabaseプロジェクト

本番データを一切汚したくない場合は、別のSupabaseプロジェクトを新規作成し`supabase/migrations/*.sql`を通し番号順に流し込み、`apps/mobile/.env`（必要なら`apps/web/.env.local`も）を差し替える方法もある。ただしマイグレーション約25本の再実行が必要でセットアップの手間が大きいため、通常は上記のテストアカウント方式を優先する。
